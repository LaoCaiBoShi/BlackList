## 2. 详细优化措施

### 2.1 动态批处理大小

**实现方法**：
- 根据系统CPU核心数和文件数量动态调整线程数
- 计算最佳批处理大小，平衡并行度和线程开销

**技术原理**：
- 使用`std::thread::hardware_concurrency()`获取系统CPU核心数
- 线程数设置为CPU核心数和文件数的最小值，确保充分利用系统资源
- 批处理大小根据线程数和文件数动态计算，避免线程过度竞争

**性能提升**：
- 自适应调整线程数，避免线程过多导致的上下文切换开销
- 充分利用系统CPU资源，提高处理速度

### 2.2 内存池

**实现方法**：
- 创建内存池类，预分配内存块
- 实现内存的分配和释放管理

**技术原理**：
- 内存池预先分配固定大小的内存块
- 当需要内存时，从内存池中获取，而不是每次都调用`malloc`
- 释放内存时，将内存块返回内存池，而不是直接释放
- 减少内存分配和释放的系统调用开销

**性能提升**：
- 减少内存分配和释放的开销
- 降低内存碎片
- 提高内存使用效率

### 2.3 线程池

**实现方法**：
- 创建线程池类，管理一组工作线程
- 实现任务队列和任务分配机制

**技术原理**：
- 线程池在初始化时创建固定数量的工作线程
- 工作线程从任务队列中获取任务执行
- 避免了频繁创建和销毁线程的开销
- 提供任务提交接口，支持异步任务执行

**性能提升**：
- 避免线程创建和销毁的开销
- 提高线程利用率
- 支持并行处理多个任务

### 2.4 高效数据结构

**实现方法**：
- 在BlacklistChecker中实现布隆过滤器
- 使用分层存储结构存储黑名单数据

**技术原理**：
- 布隆过滤器用于快速判断卡片是否可能在黑名单中，减少不必要的查询
- 分层存储结构：1-4位分片 -> 卡片类型(22/23) -> 有序CardInfo数组
- 对卡片信息进行位压缩存储，减少内存使用

**性能提升**：
- 提高黑名单查询速度
- 减少内存使用
- 支持高效的插入和删除操作

### 2.5 并行处理

**实现方法**：
- 使用线程池对压缩文件的解压缩和JSON文件的加载进行并行处理

**技术原理**：
- 对嵌套的ZIP文件进行并行解压缩
- 对解压后的JSON文件进行并行加载和解析
- 使用互斥锁保护共享资源，确保线程安全

**性能提升**：
- 显著提高处理速度，特别是在多核心系统上
- 充分利用系统资源，减少处理时间

```cpp
// 动态确定线程数：根据CPU核心数和文件数量
size_t numCores = std::thread::hardware_concurrency();
if (numCores == 0) numCores = 4; // 默认值

// 动态调整线程数，最多不超过文件数和CPU核心数的最小值
size_t threadNum = std::min(numCores, zipFiles.size());
if (threadNum < 1) threadNum = 1;

// 动态确定批处理大小
size_t batchSize = 1;
if (threadNum > 0) {
    batchSize = std::max(size_t(1), zipFiles.size() / threadNum);
}
```

### 4.2 内存池实现

```cpp
void* MemoryPool::allocate() {
    std::lock_guard<std::mutex> lock(mutex);
    
    if (freeBlocks.empty()) {
        char* newBlock = static_cast<char*>(std::malloc(blockSize * blocksPerChunk));
        if (!newBlock) {
            return nullptr;
        }
        
        blocks.push_back(newBlock);
        
        for (size_t i = 0; i < blocksPerChunk; ++i) {
            freeBlocks.push_back(newBlock + i * blockSize);
        }
    }
    
    void* ptr = freeBlocks.back();
    freeBlocks.pop_back();
    return ptr;
}

void MemoryPool::deallocate(void* ptr) {
    if (!ptr) {
        return;
    }
    
    std::lock_guard<std::mutex> lock(mutex);
    freeBlocks.push_back(static_cast<char*>(ptr));
}
```
