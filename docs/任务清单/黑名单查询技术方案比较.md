# 32位卡片黑名单查询技术方案

## 一、技术方案分析

### 1. 标准Bitmap方案
- **原理**：使用位数组，每个位代表一个卡片号的状态（0/1）
- **实现**：32位卡片号直接作为数组索引
- **空间复杂度**：512MB（2^32位 = 4GB/8 = 512MB）
- **时间复杂度**：O(1)
- **优点**：查询速度极快，实现简单
- **缺点**：固定内存占用，无论黑名单大小

### 2. 压缩Bitmap方案
- **原理**：使用RLE（Run-Length Encoding）或其他压缩算法
- **实现**：只存储连续的0或1的长度
- **空间复杂度**：取决于黑名单密度，可能小于512MB
- **时间复杂度**：O(log n)
- **优点**：内存利用率高
- **缺点**：查询速度稍慢，实现复杂

### 3. 哈希表方案
- **原理**：使用哈希表存储黑名单卡片号
- **实现**：卡片号作为键，值为布尔值
- **空间复杂度**：取决于黑名单大小，每个元素约4-8字节
- **时间复杂度**：O(1)
- **优点**：内存利用率高，只存储黑名单数据
- **缺点**：哈希冲突处理复杂，查询速度略慢于Bitmap

### 4. 布隆过滤器方案
- **原理**：使用多个哈希函数将卡片号映射到位数组
- **实现**：多个哈希函数计算索引，设置对应位
- **空间复杂度**：取决于误判率，通常比哈希表小
- **时间复杂度**：O(k)，k为哈希函数数量
- **优点**：内存占用低，查询速度快
- **缺点**：存在误判率，无法删除元素

### 5. 分块Bitmap方案
- **原理**：将32位空间分成多个块，只加载使用的块
- **实现**：按需加载块到内存
- **空间复杂度**：根据实际使用情况动态分配
- **时间复杂度**：O(1)
- **优点**：内存利用率高，查询速度快
- **缺点**：实现复杂度较高

```python
        return (self.bitmap[byte_idx] & (1 << bit_offset)) != 0
```

**原理**：使用Run-Length Encoding（游程编码）压缩连续的0或1序列
**实现**：

```cpp
        // 按省份分组

            std::sort(entry.second.begin(), entry.second.end());

    // 添加到黑名单

        uint16_t provinceCode = getProvinceCode(cardId);
        auto& provinceList = provinceMap[provinceCode];
        auto it = std::lower_bound(provinceList.begin(), provinceList.end(), cardId);
        if (it == provinceList.end() || *it != cardId) {
            provinceList.insert(it, cardId);
        }

        uint16_t provinceCode = getProvinceCode(cardId);
        auto it = provinceMap.find(provinceCode);
        if (it == provinceMap.end()) {
            return false; // 该省份无黑名单记录
        }
        return std::binary_search(it->second.begin(), it->second.end(), cardId);
    }
```

## 核心实现

在 `isBlacklisted` 方法中，二分查找是通过 C++ 标准库的 `std::binary_search` 函数实现的。具体代码如下：


## 实现原理

1. **省份定位**：首先提取卡片号的前4位作为省份代码，在 `provinceMap` 中查找对应的子黑名单
2. **有序性保证**：每个省份的子黑名单（`provinceList`）在添加元素时保持有序
3. **二分查找**：使用 `std::binary_search` 在有序的子黑名单中查找目标卡片号

### 有序性保证

子黑名单的有序性是在添加元素时通过 `std::lower_bound` 实现的：

```cpp
void BlacklistChecker::add(unsigned int cardId) {
    unsigned short provinceCode = getProvinceCode(cardId);
    auto& provinceList = provinceMap[provinceCode];
    
    // 使用二分查找找到插入位置

    if (it == provinceList.end() || *it != cardId) {
        provinceList.insert(it, cardId);
    }
}
```

- `std::lower_bound` 返回第一个大于或等于目标值的元素位置
- 通过在该位置插入新元素，保证了 `provinceList` 始终有序

## 二分查找的工作原理

`std::binary_search` 的工作原理如下：
1. **初始化**：设置搜索范围的左边界（`begin`）和右边界（`end`）
2. **循环**：
   - 计算中间位置 `mid = begin + (end - begin) / 2`
   - 比较中间元素与目标值：
     - 如果相等，返回 `true`
