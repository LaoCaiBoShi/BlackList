#### 3. CardInfo 位压缩存储

```cpp
struct CardInfo {
    unsigned short year_week;     // 2字节 = 16位
    unsigned long long innerId;   // 8字节 = 64位
};
```

| 存储方式       | 每条记录大小 | 节省比例 |
| -------------- | ------------ | -------- |
| **原始字符串** | 20 字节      | -        |
| **压缩后**     | 10 字节      | **50%**  |

**BCD编码优化**:
- innerId: 每位数字用4位存储（0-9）
- year_week: 高12位存year，低4位存week

#### 4. 多层索引结构

```cpp

```

| 层级       | Key              | 说明               |
| ---------- | ---------------- | ------------------ |
| **第一层** | prefix (1-4位)   | 省份+运营商        |
| **第二层** | type (9-10位)    | 卡类型 22/23       |
| **第三层** | vector<CardInfo> | 有序数组，二分查找 |

**卡号结构解析**:
```

      │   │  │  │  │  │      │
      │   │  │  │  │  └─── 11-20位: 内部编号 (innerId)
      │   │  │  │  └─── 7-8位: 星期数 (week)

      │   └─── 9-10位: 卡类型 (type)
      └───────── 1-4位: 省份+运营商 (prefix)
```

#### 6. 查询流程

```cpp
bool isBlacklisted(const std::string& cardId) {
    // 1. 布隆过滤器预检 (O(1))
    if (!bloomFilter.contains(cardId)) {
        return false;  // 快速排除
    }
    
    // 2. 提取前缀和类型
    unsigned short prefix = getPrefixCode(cardId);
    unsigned short type = getCardType(cardId);
    
    // 3. 二分查找 (O(log n))
    auto& vec = cardMap[prefix][type];
    return std::binary_search(vec.begin(), vec.end(), targetCardInfo);
}
```

#### 1. 并行处理优化

| 优化项           | 优化前  | 优化后            | 效果       |

```

省份3 ─┤

省份N ─┘
  ↓
每省内: JSON1 ─┐
             ├── 8线程Consumer模式
       JSONN ─┘
```

#### 2. 锁优化

```cpp
void addBatch(const std::vector<std::string>& cardIds) {
    // 预处理：按prefix和type分组
    std::unordered_map<unsigned short, std::unordered_map<unsigned short, std::vector<CardInfo>>> localMap;
    
    // 一次性加锁，批量合并
    std::lock_guard<std::mutex> lock(cardMapMutex);
    for (auto& prefixEntry : localMap) {
        for (auto& typeEntry : prefixEntry.second) {
            cardMap[prefixEntry.first][typeEntry.first].insert(
                cardMap[prefixEntry.first][typeEntry.first].end(),
                typeEntry.second.begin(),
                typeEntry.second.end()
            );
        }
    }
}
```

| 优化点         | 优化前         | 优化后     |
| -------------- | -------------- | ---------- |
| **锁获取次数** | N次 (N=卡数量) | **1次**    |
| **插入方式**   | 逐个push_back  | 批量insert |

#### 3. 内存优化

| 优化项                 | 说明                    |
| ---------------------- | ----------------------- |
| **MemoryPool**         | 减少频繁malloc/free开销 |
| **预分配容量**         | reserve() 避免动态扩容  |
| **布隆过滤器延迟重建** | 加载完成后统一重建      |

### ThreadSafeQueue 实现

```cpp
class ThreadSafeQueue {
    std::queue<T> queue;
    std::mutex mutex;
    std::condition_variable notEmpty;
    
public:
    void push(T item) {
        std::lock_guard<std::mutex> lock(mutex);
        queue.push(std::move(item));
        notEmpty.notify_one();
    }
    
    T pop() {
        std::unique_lock<std::mutex> lock(mutex);
        notEmpty.wait(lock, [this]() { return !queue.empty(); });
        T item = std::move(queue.front());
        queue.pop();
        return item;
    }
};
```

### 并行效果

| 组件       | 数量      | 说明            |
| ---------- | --------- | --------------- |
| **生产者** | 1个       | 解压 + 扫描文件 |
| **消费者** | CPU核数个 | 并行解析JSON    |
| **队列**   | 1个       | 解耦生产和消费  |

**优势**: 
- 生产者和消费者并行执行
- 消费者无需等待所有文件解压完成
- 充分利用多核CPU并行解析JSON
