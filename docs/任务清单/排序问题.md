## sortAll() 作用分析

### 当前实现
```cpp
void BlacklistChecker::sortAll() {
    // 遍历所有省份分片 (MAX_PROVINCE_CODE = 100)
    for (size_t shardIdx = 0; shardIdx < MAX_PROVINCE_CODE; ++shardIdx) {
        for (size_t typeIdx = 0; typeIdx < 3; ++typeIdx) {
            std::sort(cards.begin(), cards.end());  // 对该分片排序
        }
    }
}
```

### 作用范围

| 问题                                   | 答案                                          |
| -------------------------------------- | --------------------------------------------- |
| **是对分省数据单独排序吗？**           | ✅ 是的，每个省份分片单独排序                  |
| **是对全部省份吗？**                   | ✅ 是的，遍历所有100个省份分片                 |
| **是批量添加到存储结构就排序一次吗？** | ❌ **不是！** 每处理**一个JSON文件**就排序一次 |

### 严重性能问题

| 处理文件数 | sortAll()调用次数 | 每次排序分片数                  | 总排序次数                 |
| ---------- | ----------------- | ------------------------------- | -------------------------- |
| 21个JSON   | 21次              | 100个省份 × 3类型 = 300个vector | 21 × 300 = **6,300次排序** |

### 正确做法

```
处理第1个JSON (1000条) → addBatch(1000条) ← 不排序
处理第2个JSON (1000条) → addBatch(1000条) ← 不排序
...
处理第21个JSON (1000条) → addBatch(1000条) ← 不排序
    ↓
所有数据加载完成后
    ↓
sortAll() ← 只排序一次！
```

## 优化方案：按省份独立排序

### 当前方案的问题
- **全局排序**：每次处理完一个JSON就对所有省份排序
- **排序粒度太粗**：不需要每次都排所有省份

### 您的优化思路
```
省份44 (20个JSON)
    ↓
处理完第1个JSON → addBatch → 不排序
处理完第2个JSON → addBatch → 不排序
...
处理完第20个JSON → addBatch → ✅ 对省份44排序
    ↓
省份11 (1个JSON)
    ↓
处理完第1个JSON → addBatch → ✅ 对省份11排序
```

### 技术实现

**1. 跟踪每个省份的JSON数量和已处理数量**

```cpp
// 每个省份的统计信息
struct ProvinceStats {
    std::atomic<size_t> totalJsons;    // 该省份JSON总数
    std::atomic<size_t> processedJsons; // 已处理的JSON数量
};

// 全局省份统计
std::unordered_map<int, ProvinceStats> provinceStats;
```

**2. 解压线程记录JSON数量**

```cpp
size_t jsonCount = jsonFiles.size();
provinceStats[provinceInfo.provinceCode].totalJsons = jsonCount;
```

**3. 消费者线程处理完成后检查是否排序**

```cpp
// 处理JSON文件
processSingleJsonFileBatch(jsonPath, checker, counter);

// 获取省份代码（从文件路径解析）
int provinceCode = extractProvinceCode(jsonPath);

// 增加已处理计数
size_t processed = ++provinceStats[provinceCode].processedJsons;

// 检查是否全部处理完成
if (processed == provinceStats[provinceCode].totalJsons) {
    // 对该省份排序
    checker.sortProvince(provinceCode);
}
```

**4. 添加省份排序方法**

```cpp
void BlacklistChecker::sortProvince(int provinceCode) {
    if (provinceCode < 0 || provinceCode >= MAX_PROVINCE_CODE) return;
    
    for (size_t typeIdx = 0; typeIdx < 3; ++typeIdx) {
        auto& cards = provinceShards[provinceCode].cards[typeIdx];
        if (cards.size() > 1) {
            std::sort(cards.begin(), cards.end());
        }
    }
}
```

### 效果对比

| 方案         | 排序次数 | 排序范围            | 复杂度               |
| ------------ | -------- | ------------------- | -------------------- |

2. **排序范围最小化**：只排序完成的省份
