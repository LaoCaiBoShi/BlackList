# 内存流式处理技术方案及异常处理

## 一、内存流式处理架构

### 1.1 核心流程

```
ZIP文件 → 内存缓冲区 → 逐个JSON解析 → 分布式存储
           ↓
    ┌──────────────────┐
    │   内存池管理      │  减少malloc/free开销
    └──────────────────┘
           ↓
    ┌──────────────────┐
    │  多级队列缓冲     │  生产者-消费者解耦
    └──────────────────┘
```

### 1.2 关键数据结构

```cpp
struct JsonTask {
    std::vector<char> content;      // JSON内容（内存持有）
    std::string provinceCode;        // 省份代码
    std::string fileName;           // 原始文件名

struct ProvinceStats {
    std::atomic<size_t> totalJsons{0};
    std::atomic<size_t> processedJsons{0};
```

## 二、可能出现的异常及针对性措施

### 2.1 内存相关异常

| 异常类型         | 触发场景           | 影响               | 应对措施                                  |
| ---------------- | ------------------ | ------------------ | ----------------------------------------- |
| **内存分配失败** | 大文件/碎片化      | 程序崩溃           | 预设内存上限(MAX_MEMORY_MB)，超限拒绝加载 |
| **内存泄漏**     | JsonTask未正确释放 | 内存持续增长       | 使用智能指针/内存池，任务完成后立即释放   |
| **内存碎片化**   | 大量小对象分配     | 分配失败但总内存够 | 内存池预分配，固定大小块                  |
| **OOM Killer**   | 超过系统限制       | 进程被杀死         | 监控内存使用，设置水位线                  |

**代码示例：**
```cpp
// 内存限制检查
constexpr size_t MAX_MEMORY_BYTES = 1000ULL * 1024 * 1024; // 1000MB

void checkMemoryLimit(size_t needed) {
    size_t currentUsage = getCurrentMemoryUsage();
    if (currentUsage + needed > MAX_MEMORY_BYTES) {
        throw std::runtime_error("Memory limit exceeded");
    }
}

// 内存池预分配
class MemoryPool {
    static constexpr size_t BLOCK_SIZE = 64 * 1024; // 64KB块
    std::vector<char*> freeBlocks;
    
    char* allocate() {
        if (!freeBlocks.empty()) {
            char* block = freeBlocks.back();
            freeBlocks.pop_back();
            return block;
        }
        return new char[BLOCK_SIZE];
    }
    
    void deallocate(char* block) {
        freeBlocks.push_back(block);
    }
};
```

### 2.2 JSON解析异常

| 异常类型         | 触发场景            | 影响          | 应对措施                     |
| ---------------- | ------------------- | ------------- | ---------------------------- |
| **JSON格式错误** | 文件损坏/非标准格式 | 单文件跳过    | 捕获异常，记录错误，继续处理 |
| **字符编码问题** | 非UTF-8编码         | 解析乱码/崩溃 | 强制转换编码，失败则跳过     |
| **字段缺失**     | 数据不完整          | 解析结果异常  | 防御性读取，提供默认值       |
| **数组越界**     | 数据结构不符预期    | 读取崩溃      | 边界检查，安全取值           |

**代码示例：**
```cpp
bool processJsonContentFromMemory(const JsonTask& task, ProvinceStats& stats) {
    // 1. 编码检查
    if (!isValidUTF8(task.content.data(), task.content.size())) {
        LOG_ERROR("Invalid UTF-8: " << task.fileName);
        return false;
    }
    
    // 2. 安全解析
    Document doc;
    doc.Parse(task.content.data(), task.content.size());
    
    if (doc.HasParseError()) {
        LOG_ERROR("JSON parse error in " << task.fileName 
                  << " at offset " << doc.GetErrorOffset());
        return false;
    }
    
    // 3. 防御性字段读取
    if (!doc.IsArray()) {
        LOG_ERROR("Expected array in " << task.fileName);
        return false;
    }
    
    // 4. 安全遍历
    for (SizeType i = 0; i < doc.Size(); ++i) {
        const Value& item = doc[i];
        
        // 安全获取cardNo字段
        if (!item.HasMember("cardNo") || !item["cardNo"].IsString()) {
            LOG_WARN("Missing cardNo at index " << i);
            continue;
        }
        
        // ... 处理数据
    }
    
    return true;
}
```

### 2.3 多线程同步异常

| 异常类型      | 触发场景              | 影响         | 应对措施                |
| ------------- | --------------------- | ------------ | ----------------------- |
| **死锁**      | 锁顺序不一致/忘记解锁 | 线程卡死     | 避免嵌套锁，使用RAII    |
| **竞态条件**  | 非原子操作            | 数据不一致   | 使用原子变量/锁保护     |
| **线程泄露**  | 异常未传播            | 线程池枯竭   | 线程入口用try-catch包裹 |
| **队列满/空** | 生产消费速度不匹配    | 阻塞或忙等待 | 有界队列+超时机制       |

**代码示例：**
```cpp
// 线程安全队列（带超时）
template<typename T>
class ThreadSafeQueue {
    std::queue<T> queue_;
    std::mutex mutex_;
    std::condition_variable cv_;
    size_t maxSize_;
    
public:
    bool push(T&& item, std::chrono::milliseconds timeout) {
        std::unique_lock<std::mutex> lock(mutex_);
        if (!cv_.wait_for(lock, timeout, [this] { 
            return queue_.size() < maxSize_; 
        })) {
            return false; // 超时
        }
        queue_.push(std::move(item));
        cv_.notify_one();
        return true;
    }
    
    bool pop(T& item, std::chrono::milliseconds timeout) {
        std::unique_lock<std::mutex> lock(mutex_);
        if (!cv_.wait_for(lock, timeout, [this] { 
            return !queue_.empty(); 
        })) {
            return false; // 超时
        }
        item = std::move(queue_.front());
        queue_.pop();
        return true;
    }
};

// 线程入口异常处理
void consumerWorker(ThreadSafeQueue<JsonTask>& queue, ...) {
    try {
        while (true) {
            JsonTask task;
            if (!queue.pop(task, std::chrono::seconds(1))) {
                continue;
            }
            if (task.content.empty() && task.fileName.empty()) {
                break; // 终止信号
            }
            processJsonContentFromMemory(task, ...);
        }
    } catch (const std::exception& e) {
        LOG_ERROR("Consumer exception: " << e.what());
    }
}
```

### 2.5 数据一致性异常

| 异常类型             | 触发场景         | 影响         | 应对措施          |
| -------------------- | ---------------- | ------------ | ----------------- |
| **省份数据不完整**   | 部分JSON处理失败 | 查询结果错误 | 记录成功/失败计数 |
| **排序失败**         | 内存不足         | 数据无序     | 备选线性查找      |
| **布隆过滤器不一致** | 数据更新期间查询 | 误判/漏判    | 版本号校验        |

**代码示例：**
```cpp
struct ProvinceStats {
    size_t totalJsons = 0;
    std::atomic<size_t> processedJsons{0};
    std::atomic<size_t> failedJsons{0};
    std::vector<CardInfo> cards;
    std::atomic<bool> sortComplete{false};
};

// 处理完成后校验
bool finalizeProvince(ProvinceStats& stats) {
    if (stats.processedJsons.load() != stats.totalJsons) {
        LOG_ERROR("Province processing incomplete: " 
                  << stats.processedJsons << "/" << stats.totalJsons);
        return false;
    }
    
    if (!stats.sortComplete.load()) {
        std::sort(stats.cards.begin(), stats.cards.end(), 
                  [](const auto& a, const auto& b) {
                      return a.cardNo < b.cardNo;
                  });
        stats.sortComplete = true;
    }
    
    return true;
}
```

## 三、异常处理策略总结

```
┌─────────────────────────────────────────────────────────┐
│                    异常处理层级                           │
├─────────────────────────────────────────────────────────┤
│  L1. 预防    │ 内存池、预分配、限制检查                   │
│  L2. 检测    │ 边界检查、返回值验证、状态监控             │
│  L3. 恢复    │ 重试、降级、跳过                           │
│  L4. 报告    │ 日志记录、错误分类、度量上报               │
└─────────────────────────────────────────────────────────┘
```

**关键原则：**
1. **Fail Fast** - 启动时检查，失败立即报错
2. **Graceful Degradation** - 部分失败不影响其他功能
3. **Resource Cleanup** - RAII确保资源释放
4. **Observability** - 充分的日志和指标
