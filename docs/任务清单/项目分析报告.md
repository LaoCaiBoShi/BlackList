## 项目功能

### 1. 黑名单检查器
- **核心功能**：检查20位卡号是否在黑名单中
- **存储结构**：使用三级索引结构
  - 第一级：1-4位分片（省份+运营商）
  - 第二级：卡片类型（9-10位）
  - 第三级：有序CardInfo数组（使用二分查找）
- **数据压缩**：CardInfo结构使用位压缩存储，节省内存
  - year_week：16位存储年份和星期数
  - innerId：64位存储11-20位内部编号

### 2. 加载功能
- **支持的文件格式**：
  - JSON文件（通过`loadFromJsonFile`）
  - 文本文件（通过`loadTxtBlacklist`）
  - 压缩文件（通过`loadBlacklistFromCompressedFile`，但目前只是解压，需要手动加载）
- **内存管理**：
  - 检查系统剩余内存
  - 计算当前黑名单占用的内存
  - 支持并行加载和原子性替换

### 3. 查询功能
- **交互式查询**：命令行界面，支持输入20位卡号进行查询
- **快速查询**：使用二分查找，时间复杂度O(log n)
- **输入验证**：检查卡号长度和格式

### 4. Qt GUI界面
- **网络功能**：
  - 服务器启动和监听
  - 客户端连接和消息发送
- **文件功能**：
  - 文件浏览和打开
  - 文本文件内容显示
- **网络请求**：
  - URL加载和内容显示

## 技术特点

1. **内存优化**：
   - 使用位压缩存储卡片信息
   - 三级索引结构减少查询时间
   - 内存使用监控

2. **性能优化**：
   - 二分查找加速查询
   - 并行加载新黑名单
   - 原子性替换旧黑名单

3. **容错处理**：
   - 输入验证
   - 异常捕获
   - 无效卡号过滤

4. **跨平台**：
   - 支持Windows和Linux平台
   - 条件编译处理平台差异

## 编译分析

### 编译环境需求
- C++11或更高版本的编译器
- （可选）Qt5库（用于GUI界面）

### 缺失文件
- `server.h` - 在mainwindow.h中引用，但不存在
- `client.h` - 在mainwindow.h中引用，但不存在
- `ui_mainwindow.h` - 在mainwindow.cpp中引用，但不存在（Qt UI生成文件）

### 编译建议

#### 方法1：使用g++编译器
```bash
g++ main.cpp blacklist_checker.cpp -o blacklist_checker.exe
g++ test_blacklist.cpp blacklist_checker.cpp -o test_blacklist.exe
g++ test.cpp -o test.exe
```

#### 方法2：使用CMake
1. 确保安装了CMake

```bash
cmake --build .
```
