## 黑名单数据结构分析

### 📊 整体架构

```
BlacklistChecker
├── cardMap (主存储)
│   └── prefix (1-4位)
│       └── type (9-10位)
│           └── vector<CardInfo> (有序数组)
├── bloomFilter (布隆过滤器)
└── versionInfo (版本信息)
```

### 1️⃣ CardInfo 结构体 - 位压缩存储

```cpp
struct CardInfo {
    unsigned short year_week;     // 2字节 = 16位
    unsigned long long innerId;   // 8字节 = 64位
};
```

**设计原理**：
```cpp
CardInfo(unsigned short y, unsigned short w, const std::string& innerIdStr) {
    // year_week: 高12位存year，低4位存week
    year_week = ((y / 10) << 12) | ((y % 10) << 8) | ((w / 10) << 4) | (w % 10);
    
    // innerId: 每位数字用4位存储（BCD编码）
    innerId = 0;
    for (size_t i = 0; i < 10; i++) {
        if (i < innerIdStr.length()) {
            int digit = innerIdStr[i] - '0';
            innerId |= (static_cast<unsigned long long>(digit) << (i * 4));
        }
    }
}
```

**优势**：
| 指标       | 原始存储 | 优化后      | 节省    |
| ---------- | -------- | ----------- | ------- |
| 卡号字符串 | 20 字节  | 10 字节     | **50%** |
| year_week  | -        | 2 字节      | -       |
| innerId    | 10 字节  | 8 字节      | **20%** |
| **总计**   | 20 字节  | **10 字节** | **50%** |

### 2️⃣ 多层索引结构 - `cardMap`

```cpp
std::unordered_map<unsigned short, std::unordered_map<unsigned short, std::vector<CardInfo>>> cardMap;
```

**设计原理**：
- **第一层 key**: `prefix` (1-4位，省份+运营商)
- **第二层 key**: `type` (9-10位，卡类型 22/23)
- **第三层 value**: `vector<CardInfo>` (有序数组)

**卡号结构解析**：
```
卡号: 4401 22 23 06 20 0008 31351
      │   │  │  │  │  │      │
      │   │  │  │  │  └─── 11-20位: 内部编号 (innerId)
      │   │  │  │  └─── 7-8位: 星期数 (week)
      │   │  │  └─── 5-6位: 年份 (year)
      │   └─── 9-10位: 卡类型 (type)
      └───────── 1-4位: 省份+运营商 (prefix)
```

**优势**：
1. **查询路径优化**：O(1) 定位到具体前缀和类型
2. **范围缩小**：每个叶子节点数据量大幅减少
3. **内存局部性**：同类型数据集中存储，缓存友好

### 3️⃣ 布隆过滤器 - 快速预检

```cpp
BloomFilter(size_t expectedElements = 1000000, double falsePositiveRate = 0.001)
```

**设计原理**：
- **位数计算**：m = -n * ln(p) / (ln(2)²)
- **哈希函数数**：k = m/n * ln(2)

**优势**：
| 特性          | 说明                         |
| ------------- | ---------------------------- |
| **O(1) 预检** | 快速判断"一定不在"或"可能在" |
| **空间高效**  | 1亿位 ≈ 12.5MB               |
| **零漏检**    | 不存在的一定返回false        |
| **容错性**    | 允许一定误判率（0.1%）       |

### 4️⃣ 二分查找 - 精确匹配

```cpp
bool isBlacklisted(const std::string& cardId) {
    // 1. 布隆过滤器预检
    if (!bloomFilter.contains(cardId)) {
        return false;  // 快速排除
    }
    
    // 2. 精确定位
    unsigned short prefix = getPrefixCode(cardId);
    unsigned short type = getCardType(cardId);
    
    // 3. 二分查找
    auto& vec = cardMap[prefix][type];
    return std::binary_search(vec.begin(), vec.end(), targetCardInfo);
}
```

**查询复杂度**：
| 阶段         | 复杂度       | 说明              |
| ------------ | ------------ | ----------------- |
| 布隆过滤器   | O(1)         | 7次哈希运算       |
| 定位索引     | O(1)         | unordered_map查找 |
| 二分查找     | O(log n)     | n = 该类型卡数量  |
| **总复杂度** | **O(log n)** | 非常高效          |

### 5️⃣ 批量处理优化 - 减少锁竞争

```cpp
void addBatch(const std::vector<std::string>& cardIds) {
    // 预处理：按prefix和type分组
    std::unordered_map<unsigned short, std::unordered_map<unsigned short, std::vector<CardInfo>>> localMap;
    
    // 一次性加锁，批量合并
    std::lock_guard<std::mutex> lock(cardMapMutex);
    for (auto& prefixEntry : localMap) {
        for (auto& typeEntry : prefixEntry.second) {
            cardMap[prefixEntry.first][typeEntry.first].insert(
                cardMap[prefixEntry.first][typeEntry.first].end(),
                typeEntry.second.begin(),
                typeEntry.second.end()
            );
        }
    }
}
```

**优势**：
- 锁获取次数从 **N次** 减少到 **1次**（N=卡数量）
- 数据合并使用 `insert` 比逐个 `push_back` 更高效
