# 线程数量设计技术方案

## 设计原则

| 原则              | 说明                                        |
| ----------------- | ------------------------------------------- |
| **CPU核心利用率** | 线程数 ≤ CPU核心数（避免上下文切换）        |
| **任务特性**      | IO密集型线程可以多一些，CPU密集型需要少一些 |
| **内存限制**      | 每个线程需要一定的内存空间                  |
| **动态平衡**      | 解压速度 ≈ 消费速度                         |

---

## 线程类型分析

| 线程类型       | 任务特性  | 影响因素                         |
| -------------- | --------- | -------------------------------- |
| **解压线程**   | IO密集型  | 磁盘速度、网络速度（如果是远程） |
| **消费者线程** | CPU密集型 | CPU解析速度、SIMD加速            |

## 线程数量设计

### 1. 基础配置公式

```cpp
struct ThreadPoolConfig {
    size_t extractThreadCount;  // 解压线程数
    size_t consumerThreadCount; // 消费者线程数
    size_t totalThreads;        // 总线程数
};

/**
 * @brief 计算最优线程配置
 */
ThreadPoolConfig calculateOptimalThreads(size_t cpuCount, size_t memoryMB, bool isSsd) {
    ThreadPoolConfig config;
    
    // 公式：totalThreads = cpuCount（充分利用CPU）
    // 公式：extractRatio = isSsd ? 0.25 : 0.15（SSD可支持更多IO线程）
    
    if (isSsd) {
        // SSD配置：解压25%，消费75%
        config.extractThreadCount = cpuCount / 4;
        config.consumerThreadCount = cpuCount - config.extractThreadCount;
    } else {
        // HDD配置：解压15%，消费85%
        config.extractThreadCount = std::max(2UL, cpuCount / 6);
        config.consumerThreadCount = cpuCount - config.extractThreadCount;
    }
    
    // 确保最小值
    config.extractThreadCount = std::max(2UL, config.extractThreadCount);
    config.consumerThreadCount = std::max(4UL, config.consumerThreadCount);
    
    config.totalThreads = config.extractThreadCount + config.consumerThreadCount;
    
    return config;
}
```

## 动态调整策略

### 速度匹配公式

```
解压速度（JSON/秒） ≈ 消费速度（JSON/秒）

理想情况：
- 解压线程产生任务的速度 ≈ 消费者处理任务的速度
- 队列不会积压（内存不膨胀）
- 队列不会为空（线程不空闲）
```

### 动态调整算法

```cpp
class DynamicThreadBalancer {
    std::atomic<size_t> extractThreads;
    std::atomic<size_t> consumerThreads;
    
    // 监控指标
    std::atomic<size_t> queueSize{0};           // 当前队列大小
    std::atomic<size_t> emptyCount{0};           // 队列为空的次数
    std::atomic<size_t> fullCount{0};            // 队列满的次数
    
    // 阈值
    static constexpr size_t QUEUE_EMPTY_THRESHOLD = 5;   // 队列空N次
    static constexpr size_t QUEUE_FULL_THRESHOLD = 5;    // 队列满N次
    
public:
    /**
     * @brief 根据队列状态动态调整线程数
     */
    void adjust() {
        // 如果队列长时间为空，增加解压线程，减少消费者
        if (emptyCount > QUEUE_EMPTY_THRESHOLD) {
            if (consumerThreads > 4) {
                consumerThreads--;
                extractThreads++;
                emptyCount = 0;
                std::cout << "Adjusting: +1 extract, -1 consumer (queue often empty)" << std::endl;
            }
        }
        
        // 如果队列长时间满，增加解压线程
        if (fullCount > QUEUE_FULL_THRESHOLD) {
            if (extractThreads < cpuCount / 2) {
                extractThreads++;
                fullCount = 0;
                std::cout << "Adjusting: +1 extract (queue often full)" << std::endl;
            }
        }
    }
    
    /**
     * @brief 记录队列状态
     */
    void recordQueueState(size_t currentSize, size_t maxSize) {
        if (currentSize == 0) {
            emptyCount++;
        } else if (currentSize >= maxSize * 0.9) {
            fullCount++;
        } else {
            emptyCount = 0;
            fullCount = 0;
        }
    }
};
```

## 内存限制

### 每线程内存需求

| 线程类型   | 内存需求 | 说明                        |
| ---------- | -------- | --------------------------- |
| 解压线程   | ~10MB    | ZIP缓冲区、文件头等         |
| 消费者线程 | ~5MB     | simdjson parser、临时缓冲区 |
| JsonTask   | ~150KB   | 每个任务平均大小            |

### 内存约束公式

```cpp
size_t calculateMaxConsumerThreads(size_t memoryMB, size_t queueSize) {
    // 可用内存 = 总内存 - 系统占用 - 预留
    size_t availableMemory = memoryMB * 1024 * 1024;  // 转为字节
    
    // 每个消费者线程需要约5MB
    // 队列中的JsonTask总共约150KB * queueSize
    
    size_t perThreadMemory = 5 * 1024 * 1024;  // 5MB
    size_t queueMemory = 150 * 1024 * queueSize; // 150KB * 队列大小
    
    size_t maxByMemory = (availableMemory - queueMemory) / perThreadMemory;
    
    return std::max(4UL, maxByMemory); // 最少4个
}
```

## 最终配置方案

### 推荐的线程配置

```cpp
struct ThreadPoolConfig {
    size_t extractThreadCount;   // 解压线程数
    size_t consumerThreadCount;  // 消费者线程数
    size_t queueSize;            // 队列大小
    size_t maxMemoryMB;          // 最大内存占用
};

/**
 * @brief 获取推荐的线程配置
 */
ThreadPoolConfig getRecommendedConfig() {
    size_t cpuCount = std::thread::hardware_concurrency();
    size_t memoryMB = getAvailableMemoryMB() / (1024 * 1024); // 转为MB
    bool isSsd = isSsdDrive(".");
    
    ThreadPoolConfig config;
    
    // 基础配置
    if (isSsd) {
        // SSD配置
        config.extractThreadCount = std::min(8UL, cpuCount / 4);
        config.queueSize = 240; // 240个任务缓冲
        
        // 队列内存 = 240 * 150KB = 36MB
        // 消费者线程内存 = 24 * 5MB = 120MB
        // 总计约160MB，远低于1GB限制
    } else {
        // HDD配置
        config.extractThreadCount = std::min(4UL, cpuCount / 8);
        config.queueSize = 480; // HDD需要更大缓冲
        
        // 队列内存 = 480 * 150KB = 72MB
        // 消费者线程内存 = 28 * 5MB = 140MB
        // 总计约212MB
    }
    
    config.consumerThreadCount = cpuCount - config.extractThreadCount;
    config.maxMemoryMB = (config.queueSize * 150 + config.consumerThreadCount * 5) / 1024; // 转为MB
    
    return config;
}
```

### 推荐配置表

| CPU      | 磁盘    | 解压线程 | 消费线程 | 总线程 | 队列大小 | 预估内存   |
| -------- | ------- | -------- | -------- | ------ | -------- | ---------- |
| 4核      | SSD     | 1        | 3        | 4      | 60       | ~20MB      |
| 8核      | SSD     | 2        | 6        | 8      | 120      | ~40MB      |
| 16核     | SSD     | 4        | 12       | 16     | 240      | ~80MB      |
| **32核** | **SSD** | **8**    | **24**   | **32** | **240**  | **~160MB** |
| 32核     | HDD     | 4        | 28       | 32     | 480      | ~212MB     |
| 64核     | SSD     | 16       | 48       | 64     | 480      | ~320MB     |

## 实施建议

### 推荐方案（32核SSD）

```cpp
// 最终推荐的配置
config.extractThreadCount = 8;   // 解压线程（IO密集）
config.consumerThreadCount = 24; // 消费者线程（CPU密集）
config.queueSize = 240;           // 队列大小（可调整）
```

### 理由

| 理由                | 说明                       |
| ------------------- | -------------------------- |
| **IO不是瓶颈**      | SSD解压速度快              |
| **CPU需要充分利用** | 24个消费者线程充分解析JSON |
| **队列缓冲足够**    | 240个任务 ≈ 36MB缓冲       |
| **内存可控**        | 总计约160MB，远低于限制    |

---

## 总结

| 配置项         | 推荐值 | 计算公式                    |
| -------------- | ------ | --------------------------- |
| **解压线程**   | 8个    | min(8, cpuCount/4)，SSD专用 |
| **消费者线程** | 24个   | cpuCount - extractThreads   |
| **队列大小**   | 240个  | consumerThreads × 10        |
| **预估内存**   | ~160MB | 可控范围内                  |

**这个配置能够实现解压速度和消费速度的平衡，队列不会积压也不会为空。**
