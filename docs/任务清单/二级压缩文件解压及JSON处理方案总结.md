# 二级压缩文件解压及JSON处理方案总结

## 一、整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                    输入：一级ZIP文件                               │
└───────────────────────────┬─────────────────────────────────────┘
                            │ 单线程解压
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│                 阶段1：一级ZIP解压                               │
│    输出：31个省份的二级压缩文件（如 20230620_11.zip）            │
└───────────────────────────┬─────────────────────────────────────┘
                            │ 多省份并行
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│              阶段2：二级压缩文件解压（解压线程池）                │
│                                                               │
│  解压线程1 ──▶ 解压省11.zip ──▶ 收集JSON路径 ──┐              │
│  解压线程2 ──▶ 解压省44.zip ──▶ 收集JSON路径 ──┼──▶ 全局队列  │
│  解压线程3 ──▶ 解压省XX.zip ──▶ 收集JSON路径 ──┤              │
│  ...                                                    │      │
│  解压线程N ──▶ 解压省YY.zip ──▶ 收集JSON路径 ──┘              │
└───────────────────────────┬─────────────────────────────────────┘
                            │ 队列
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│              阶段3：JSON文件处理（消费者线程池）                  │
│                                                               │
│  消费者线程1 ◀── 从队列取任务 ◀── 处理JSON_1, JSON_5, ...     │
│  消费者线程2 ◀── 从队列取任务 ◀── 处理JSON_2, JSON_6, ...     │
│  ...                                                        │
│  消费者线程N ◀── 从队列取任务 ◀── 处理JSON_N, ...            │
└─────────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│              阶段4：数据排序                                     │
│    cardMap 按省份分片，每片内排序                                │

```

## 二、线程配置

### 动态线程配置（根据CPU核心数和省份数量）

| 配置项           | 32核CPU | 说明                  |
| ---------------- | ------- | --------------------- |
| **解压线程数**   | 8       | IO密集型任务，固定8个 |
| **消费者线程数** | 32      | CPU核心数，JSON处理   |
| **队列大小**     | 128     | 缓冲4批任务           |
| **批处理大小**   | 10000   | 每批次处理的卡号数量  |

### 配置计算公式

```cpp
config.extractThreads = 8;  // 固定8个
config.totalThreads = cpuCount;  // 消费者线程数=CPU核心数
config.queueSize = cpuCount * 4;  // 队列大小=核心数×4
```

## 三、关键特性

### 1. 全局任务队列
- 所有省份的JSON文件路径都放入同一个队列
- 消费者线程从队列获取任务，不区分省份
- 实现动态负载均衡

### 2. 生产者-消费者模式
- **生产者**：解压线程，负责解压ZIP并收集JSON路径
- **消费者**：处理线程，负责解析JSON并提取卡号
- 解压和处理可以流水线并行

### 3. 线程终止机制
- 每个消费者线程收到一个空字符串终止信号
- 收到终止信号立即退出，无需检查队列状态

### 4. 动态负载均衡
- 所有消费者线程竞争队列中的任务
- 哪个线程先拿到任务就先处理
- 不会出现线程闲置

## 四、处理流程

| 阶段      | 操作        | 线程数 | 并行方式             |
| --------- | ----------- | ------ | -------------------- |
| **阶段1** | 解压一级ZIP | 1      | 串行（单文件）       |
| **阶段2** | 解压二级ZIP | 8      | 多省份并行           |
| **阶段3** | 解析JSON    | 32     | 全局队列竞争         |
| **阶段4** | 数据排序    | 1      | 串行（依赖完整数据） |
